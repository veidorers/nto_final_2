# Примеры подсчёта NDCG@20

Эти примеры соответствуют определению метрики из [`task.md`](task.md).

Полезные значения discount:

- `discount(1) = 1 / log2(2) = 1`
- `discount(2) = 1 / log2(3) ≈ 0.63093`
- `discount(3) = 1 / log2(4) = 0.5`
- `discount(4) = 1 / log2(5) ≈ 0.43068`

Во всех примерах `rel(rank) = 1`, если издание на позиции `rank` является потеряшкой для пользователя, иначе `rel(rank) = 0`.

## Пример 1: все потеряшки в топе (NDCG = 1.0)

Скрытые потеряшки пользователя (истина): `{101, 202}`.

Предсказание участника (покажем только позиции с `rel = 1`; для остальных позиций 3..20 `rel = 0`):

| rank | edition_id | rel | вклад в DCG |
|-----:|-----------:|----:|------------:|
| 1 | 101 | 1 | \(1·1=1\) |
| 2 | 202 | 1 | \(1·0.63093≈0.63093\) |

Итого:

- `DCG ≈ 1 + 0.63093 = 1.63093`
- `IDCG` совпадает (идеальная выдача: обе потеряшки на первых позициях) ⇒ `IDCG ≈ 1.63093`
- `NDCG = DCG / IDCG = 1.0`

## Пример 2: потеряшки ниже по рангу (NDCG < 1.0)

Та же скрытая разметка `{101, 202}`, но обе потеряшки стоят ниже:

| rank | edition_id | rel | вклад в DCG |
|-----:|-----------:|----:|------------:|
| 2 | 101 | 1 | \(1·0.63093≈0.63093\) |
| 3 | 202 | 1 | \(1·0.5=0.5\) |

Итого:

- `DCG ≈ 0.63093 + 0.5 = 1.13093`
- `IDCG` такой же, как в примере 1: `IDCG ≈ 1.63093`
- `NDCG ≈ 1.13093 / 1.63093 ≈ 0.6937`

## Пример 3: есть промахи и непорядок в ранжировании

Скрытые потеряшки пользователя (истина): `{11, 22, 33}`.

Предсказание участника (покажем только позиции с `rel = 1`):

| rank | edition_id | rel | вклад в DCG |
|-----:|-----------:|----:|------------:|
| 1 | 11 | 1 | \(1·1=1\) |
| 4 | 22 | 1 | \(1·0.43068≈0.43068\) |

Итого:

- `DCG ≈ 1 + 0.43068 = 1.43068`

Теперь `IDCG`: идеальная выдача — три потеряшки на первых позициях:

- `IDCG ≈ discount(1) + discount(2) + discount(3) = 1 + 0.63093 + 0.5 = 2.13093`
- `NDCG ≈ 1.43068 / 2.13093 ≈ 0.6713`
