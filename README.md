# NTO AI Final Baseline

Этот репозиторий — baseline для финального кейса НТО ИИ «Потеряшки». Задача очень коротко: по неполным логам и метаданным нужно восстановить для каждого пользователя из `targets.csv` топ-20 наиболее вероятных потерянных позитивных взаимодействий с книгами.

Baseline уже содержит воспроизводимый пайплайн, локальную валидацию, контракты данных и формат сабмита. При этом он намеренно оставлен достаточно простым: наша цель не в том, чтобы дать вам “почти готовое решение”, а в том, чтобы дать удобную рамку для быстрых экспериментов и интеграции новых идей.

Проект можно запустить одной командой прямо сейчас:

```bash
uv run python -m src.platform.cli.entrypoint run --config configs/experiments/baseline.yaml
```

## Что уже реализовано в baseline

Текущий baseline идёт по stage-based пайплайну:

```text
prepare_data -> build_features -> generate_candidates -> rank_and_select -> make_submission
```

По смыслу он делает следующее:

- собирает `Dataset` из входных CSV
- строит простые признаки
- генерирует кандидатов несколькими источниками
- ранжирует кандидатов и формирует итоговый `submission.csv`
- кеширует промежуточные артефакты, чтобы не пересчитывать одинаковые этапы повторно

Архитектурно репозиторий разделён на две зоны:

- [`src/competition/`](src/competition/) — зона решения, которую вам и нужно расширять
- [`src/platform/`](src/platform/) — инфраструктура запуска, которую обычно менять не требуется

## Baseline намеренно наивный по DS, но богатый инженерно

Мы оставили несколько вещей простыми:

- генераторы и ранкер в baseline эвристические и минималистичные
- в общем пайплайне нет обучения моделей
- feature engineering в baseline показан только как отправная точка, а не как сильный production-grade набор признаков

Мы ожидаем, что вы будете добавлять, заменять и улучшать:

- новые фичи
- новые генераторы кандидатов
- более сильные ранкеры
- внешне обученные модели и их интеграцию в общий pipeline

## Как мы рекомендуем с этим работать

Baseline собран так, чтобы им было удобно пользоваться команде из двух участников.

Во-первых, разные модели и идеи можно разрабатывать по отдельности, а затем совмещать внутри общего baseline. Это удобно и для параллельной работы, и для аккуратной интеграции новых фичей, генераторов и ранкеров.

Во-вторых, в baseline нет встроенного этапа обучения. Любое обучение ваших моделей можно вынести в отдельные ноутбуки или скрипты, а в основном пайплайне уже просто подгружать готовые веса, коэффициенты, таблицы признаков или другие артефакты для генератора и ранкера. Если захотите, ранкеров и генераторов может быть несколько.

В-третьих, мы как организаторы предлагаем относиться к этому baseline как к исследовательской платформе:

- если у вас остались полезные фичи, кандидаты или ранкеры с предыдущих этапов, их имеет смысл переносить и адаптировать сюда
- мы поощряем использование ИИ-инструментов, если они бесплатны и не нарушают правила НТО: можно генерировать черновики функций, генераторов, ранкеров и feature-файлов, а затем быстро интегрировать их в `competition`

В-четвёртых, более сложный пайплайн с кешированием нужен не ради “инженерной красоты”, а ради скорости экспериментов. Если вы уже запускали какой-то этап и его входы не поменялись, baseline постарается не пересчитывать его заново или сделать повторный запуск максимально быстрым.
